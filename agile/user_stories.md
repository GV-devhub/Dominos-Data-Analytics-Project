# User Stories – Domino’s Pizza Analytics Project

This document captures user stories representing the needs of different stakeholders in a Domino’s-style pizza business. These stories guide the SQL analysis, dashboard design, and security model.

---

## 1. Store Manager

**Story 1:**  
As a **Store Manager**, I want to view **daily and monthly sales** so that I can plan staffing and inventory.

**Story 2:**  
As a **Store Manager**, I want to see the **top-selling pizzas in my store** so that I can adjust menu placement and promotions.

---

## 2. Regional Manager

**Story 3:**  
As a **Regional Manager**, I want to compare **sales across multiple stores** so that I can identify high- and low-performing locations.

**Story 4:**  
As a **Regional Manager**, I want to analyze **peak ordering hours** so that I can optimize scheduling across the region.

---

## 3. Marketing Analyst

**Story 5:**  
As a **Marketing Analyst**, I want to know which **pizza categories and sizes** generate the most revenue so that I can design targeted promotions.

**Story 6:**  
As a **Marketing Analyst**, I want to analyze **customer ordering patterns by time and day** so that I can plan campaigns effectively.

---

## 4. Executive Leadership

**Story 7:**  
As an **Executive**, I want to see **high-level KPIs** (total revenue, AOV, top products) so that I can evaluate business performance.

**Story 8:**  
As an **Executive**, I want access to a **simple, visual dashboard** so that I can quickly understand trends.

---

## 5. Data Analyst / BI Developer

**Story 9:**  
As a **Data Analyst**, I want to create **clean, standardized datasets** so that reporting is accurate and consistent.

**Story 10:**  
As a **Data Analyst**, I want to build **views and reusable SQL scripts** so that dashboards can be refreshed easily.

---

## 6. Security Officer (RBAC)

**Story 11:**  
As a **Security Officer**, I want to ensure **role-based access** so that sensitive data is only visible to authorized users.

**Story 12:**  
As a **Security Officer**, I want to enforce **least-privilege access** so that each role only sees what they need.

---

These user stories form the foundation of the Agile workflow and guide the technical implementation of the project.
